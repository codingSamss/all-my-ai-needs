# 全局配置

## 沟通与协作
- 全程中文；代码、命令、专有名词保留英文。不使用 emoji。
- 默认面向 Sam SAMA；先理解上下文与目标，质量与安全优先。
- 先分析需求/约束再动手；歧义先澄清，关键决策说明理由。
- 同一问题连续失败 3 次，停下复盘换策略。
- 高风险系统命令、破坏性操作必须先确认。

## 执行
- 先想清方案与验证手段，不要边写边试错。
- 关键逻辑（业务规则、非直观分支、协议/缓存/并发/回滚）补简短中文注释；自解释代码不堆注释。
- 收尾跑必要测试，确认改动范围与预期一致；受限时说明原因。
- 本次会话新增的冗余代码（未调用方法、无用 import/变量、调试代码）收尾清理；不动存量代码。

## 输出风格
- 直接、清楚地按正常书面逻辑编排内容；不套用「一句话结论 / TL;DR / 一句话先记住」之类的开头模板；能一句说清就别硬拆成列表。
- 3 个以上并列才用 bullet，最多两级；多维对比才用表格。
- 命令/路径/代码片段放代码块；目录结构用 ASCII tree。
- 失败按“现象 -> 原因 -> 修复”表达；避免大段文字墙。

## 浏览器
- 优先 HTTP/API，能不开浏览器就不开。
- 控制本机真实 Chrome 走 `chrome-devtools` MCP；隔离自动化走 `playwright` MCP。
- `playwright-ext` 默认不用（抢焦点，能力已被 `chrome-devtools` 覆盖）。

## 工具与项目约束
- Java 项目不擅自跑 `mvn` 或拉起项目，让用户手动启动。
- Obsidian iCloud Vault 的 `.md` 外部改动后执行 `touch <file>` 刷新时间戳。
- 知识库/日记/笔记优先走对应 skill；更新文档要整合式修订，不只在文末追加；路径不明先问。
- Git 操作优先用 `$git-ops`；不主动 push。
- 删除用 `trash`，禁用 `rm -rf`；覆盖目录用 `cp -r`，不先删后复制；单文件建议 <500 行。

## all-my-ai-needs 同步
- platform-first，仅处理受管的 Claude/Codex 内容。
- 用户只说“同步/看差异”未明确授权写入时，先只读 `check` 汇总差异，审批后再 `apply`/`commit`/`push`。
- `runtime.yaml` 不下发到运行目录；完成后汇报清单（新增/更新/删除/跳过）。

## 提交前审计
- 隐私扫描（允许 `<TOKEN>` 占位符）：
  - `git grep -nEI "AKIA[0-9A-Z]{16}|ASIA[0-9A-Z]{16}|ghp_[A-Za-z0-9]{36}|github_pat_[A-Za-z0-9_]{20,}|xox[baprs]-[A-Za-z0-9-]{10,}|sk-[A-Za-z0-9]{20,}|x-api-key\\s*[:=]\\s*\"[^<\\\"]+\""`
- `git diff --check && git diff --cached --check`；删文件后查无残留引用。

## 网络代理
- 除国内站点外默认走 `7897`：`HTTP_PROXY=http://127.0.0.1:7897`、`HTTPS_PROXY=http://127.0.0.1:7897`；需要 SOCKS 时加 `ALL_PROXY=socks5://127.0.0.1:7897`。
